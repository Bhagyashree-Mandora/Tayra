Tayra Restores CUD Operations on Collections

As a MongoDB admin
I want to be able to backup and replay Create, Update and Delete Operations from recorded oplog file to standalone MongoDB
So that I can bring it in sync with the source MongoDB from where it was backed up.

When the Replica Set is running

1. Do Some CUD Operations

Connect to MongoDB node localhost on port 27017
use database test
run 
db.things.save({name:1})
run 
db.things.save({name:2})
run 
db.things.save({name:3})
run 
db.things.remove({name:1})
run 
db.things.update({name:3}, {name:4})

2. Run Backup Utility

Open terminal and
run 
backup -s localhost -f test.out
show 
stdout
and ensure 
stdout
 contains Backup Started

3. Run Restore Utility

Open terminal and
run 
restore -d localhost -f test.out --port=27020
show 
stdout
show 
stderr
and ensure 
stdout
 contains Process started
