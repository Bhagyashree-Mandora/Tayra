Tayra Restores CUD Operations on Collections

As a MongoDB admin
I want to be able to backup and replay Create, Update and Delete Operations from backup file to standalone MongoDB
So that I can bring it in sync with the source MongoDB from where it was backed up.

When the Tayra Environment is running with Source 
localhost
 On Port 
17017
 and Target 
localhost
 On Port 
27021

1. Do Some CUD Operations

run mongo command on

db.collOne.save({name:"docThree"})

2. Run Backup Utility

Open terminal
and run 
backup -s localhost -f test.out --port=17017
and show 
stdout
and ensure 
stdout
 contains Process started

3. Run Restore Utility

Open terminal
and run 
restore -d localhost -f test.out --port=27021
and show 
stdout
and show 
stderr
and ensure 
stdout
 contains Process started
